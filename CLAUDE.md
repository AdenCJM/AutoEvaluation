# AutoEvaluation — Project Instructions

An autonomous optimisation engine that iteratively improves LLM skill files by hill-climbing on evaluation scores.

## Key Concepts

- **The loop** is driven by `program.md` via `claude -p program.md`. That file is the agent's instructions during a run — never modify it without good reason.
- **`SKILL.md`** is the file being autonomously rewritten during optimisation. Don't edit it manually while a run is in progress. `SKILL.md.best` tracks the highest-scoring version.
- **`config.yaml`** is generated by `setup.py` and must conform to the schema in `config.template.yaml`. Edit the template if you're adding new options.
- **`results.tsv`** is the experiment history (append-only during runs).

## File Ownership

| File / Dir | Mutability | Notes |
|---|---|---|
| `program.md` | Rarely changed | Loop instructions for Claude Code |
| `SKILL.md` | Auto-modified during runs | The optimisation target |
| `config.yaml` | Generated by `setup.py` | User settings — schema must match template |
| `tools/` | Stable | Deterministic scripts — don't modify during a run |
| `prompts/` | Stable | Test scenarios — don't modify during a run |
| `.tmp/` | Disposable | Intermediate samples and logs, safe to delete |
| `results.tsv` | Append-only during runs | Full experiment history |

## Tools

All scripts in `tools/` are deterministic executors. They take CLI args, do one thing, and return structured output. The main ones:

- `utils.py` — shared utilities (config loading, validation, sanitisation)
- `experiment_runner.py` — orchestrates a single eval cycle (generate → judge → score)
- `model_client.py` — LLM provider abstraction with retry, token tracking, cost estimation
- `generate_samples.py` — produces outputs from the current SKILL.md (supports parallel generation)
- `eval_llm_judge.py` — LLM-as-judge scoring (supports semi-blind mode and separate judge model)
- `eval_deterministic.py` — optional rule-based metrics
- `score_aggregator.py` — weighted composite scoring
- `run_loop.py` — standalone headless loop driver (alternative to `claude -p program.md`)
- `dashboard_server.py` — live score dashboard on localhost:8050

## Running

```bash
# First-time setup
python3 setup.py

# Run the optimisation loop
claude -p program.md

# Or headless without Claude Code
python3 tools/run_loop.py
```

## Conventions

- API keys live in `.env` only — never hardcode secrets
- `config.yaml` is the single source of truth for all settings
- One change per iteration during optimisation — keep diffs attributable
- Weights in the rubric (LLM judge + deterministic) must sum to 1.0 (auto-normalised with warning if not)
- Config is validated at startup via `utils.validate_config()` — catches missing keys and bad weights
